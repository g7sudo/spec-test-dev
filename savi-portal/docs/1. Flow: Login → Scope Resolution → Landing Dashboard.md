0. Shared Assumptions (so devs don’t guess)
	•	One web app, one domain, shared shell.
	•	Two root URL families:
	•	Platform: /platform/...
	•	Tenant: /tenant/{tenant_slug}/...
	•	After Auth (Firebase/whatever), backend exposes:
	•	roles (contains PlatformAdmin or not)
	•	tenantMemberships: [{ tenantId, tenantSlug, displayName, rolesInTenant[] }, ...]
	•	Scope dropdown:
	•	For PlatformAdmin:
	•	Platform + all tenantMemberships
	•	For TenantAdmin:
	•	Only tenantMemberships
	•	When scope changes:
	•	App hard navigates (full reload) to the appropriate URL.
	•	Example: select Sunrise Heights → browser goes to /tenant/sunrise-heights/dashboard and reloads.

1. Flow: Login → Scope Resolution → Landing Dashboard

1.1 User Journey (Both personas)

Name: F-LOGIN-01

Goal: After login, user lands on the correct dashboard with correct scope & dropdown options.

Actors:
	•	Platform Admin
	•	Tenant Admin

Steps (happy path):
	1.	User opens app root (e.g., / or /login).
	2.	User enters credentials and clicks Sign in.
	3.	App authenticates with auth provider and gets ID token.
	4.	App calls backend GET /auth/me with that token.
	5.	Backend returns:
	•	userId
	•	globalRoles: string[]
	•	tenantMemberships: array
	6.	Frontend derives:
	•	isPlatformAdmin = globalRoles.includes("PlatformAdmin")
	•	For each membership, tenantSlug etc.
	7.	Routing decision:
	•	If isPlatformAdmin === true
→ redirect to /platform/dashboard
	•	Else (no platform role, at least one tenant membership)
→ pick default tenant (e.g., first or last-used) and redirect to
/tenant/{tenant_slug}/dashboard
	8.	Browser loads that URL (full page load / hydration).
	9.	On that page:
	•	App again calls /auth/me (idempotent).
	•	Builds scope dropdown options.
	•	Renders nav + dashboard for that URL.

  └─(success)→

            ↓
        [A4] Call /auth/me (on page load)
            ↓
        [S5] Render Tenant Dashboard + Dropdown (tenants only)

2. Flow: Platform Admin – Switching to a Tenant (Gmail-style)

2.1 User Journey

Name: F-PA-SWITCH-01

Goal: Platform Admin moves from Platform view to a specific tenant, with full reload.

Preconditions:
	•	User is PlatformAdmin.
	•	User is currently on /platform/dashboard.
	•	tenantMemberships is non-empty.

Steps:
	1.	User is on /platform/dashboard.
	2.	Top bar shows Scope dropdown with:
	•	Platform (selected)
	•	List: Sunrise Heights, Green Valley, …
	3.	User clicks dropdown and selects Sunrise Heights.
	4.	Frontend handler:
	•	Computes target URL: /tenant/sunrise-heights/dashboard.
	•	Calls window.location.href = "/tenant/sunrise-heights/dashboard" (or equivalent) to force full reload.
	5.	Browser navigates to /tenant/sunrise-heights/dashboard and reloads.
	6.	On load:
	•	App calls /auth/me.
	•	Dropdown now shows Sunrise Heights as selected; options still include Platform and other tenants.
	•	Left nav switches to tenant-level navigation.
	•	Dashboard shows Sunrise Heights metrics/data.

[Start: Platform Admin is on /platform/dashboard]
  ↓
[S10] Platform Dashboard (scope = Platform)
  ↓
[A10] User clicks Scope dropdown
  ↓
[S11] Dropdown options:
     - "Platform" (selected)
     - "Sunrise Heights"
     - "Green Valley"
  ↓
[A11] User selects "Sunrise Heights"
  ↓
[A12] App computes URL: /tenant/sunrise-heights/dashboard
  ↓

     window.location.href = "/tenant/sunrise-heights/dashboard"
  ↓
[S12] Browser loads /tenant/sunrise-heights/dashboard
  ↓

  ↓
[S13] Render:
     - Scope dropdown: "Sunrise Heights" selected
     - Left nav: Tenant modules
     - Main: Tenant dashboard for "Sunrise Heights"
[End]

Edge Cases / Guards:
	•	If /auth/me on tenant page shows no membership for sunrise-heights (e.g., role revoked):
	•	Show “Access denied to this tenant.”
	•	Optionally redirect back to /platform/dashboard (for Platform Admin) or another allowed tenant.

5. URL + Reload Rules (for Devs)

To make it explicit for implementation:
	1.	Every scope change from dropdown:
	•	Compute intended URL:
	•	Platform → /platform/dashboard
	•	{tenantSlug} → /tenant/{tenantSlug}/dashboard
	•	Use hard navigation (window.location.href or normal <a href>), not client-only routing.
	•	This guarantees:
	•	Fresh /auth/me call
	•	Clean state reset per scope
	2.	Every page load on /platform/... or /tenant/...:
	•	Immediately call /auth/me.
	•	Verify user has rights for that scope.
	•	If not, redirect to a safe default (first allowed tenant or /platform/dashboard if platform admin).
